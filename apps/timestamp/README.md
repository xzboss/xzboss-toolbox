# 时间戳工具项目说明

`apps/timestamp` 是一个独立的 React 前端项目，用于时间戳识别、转换和批量扫描。项目不依赖后端，所有转换逻辑都在浏览器中完成，打包后可以作为静态站点直接运行。

## 技术栈

- React
- TypeScript
- Vite
- Tailwind CSS
- shadcn/ui 或 Radix UI
- lucide-react

## 功能需求

### 1. 单个时间戳转换

用户输入一个时间戳后，工具自动识别该值是秒级时间戳还是毫秒级时间戳，并展示识别结果。

需要展示：

- 原始输入值
- 自动识别单位：秒或毫秒
- 转换后的本地时间
- ISO 时间
- UTC 时间
- 是否为合法时间戳

识别建议：

- 10 位数字优先识别为秒级时间戳。
- 13 位数字优先识别为毫秒级时间戳。
- 允许纯数字字符串。
- 可兼容首尾空格。
- 明显超出合理时间范围的数字标记为非法。

### 2. JSON 时间戳扫描

支持用户直接粘贴 JSON 文本。工具会解析 JSON，并扫描其中所有合法时间戳，自动转换后集中展示。

需要支持：

- 对象
- 数组
- 深层嵌套结构
- 字符串形式的数字
- 数字类型的时间戳

扫描结果建议展示：

- 字段路径，例如 `user.createdAt`
- 原始值
- 识别单位：秒或毫秒
- 转换后的本地时间
- ISO 时间

JSON 解析失败时，不直接中断体验，可以提示用户当前输入不是合法 JSON，并允许用户切换到文本扫描模式。

### 3. 普通文本时间戳扫描

支持用户粘贴任意文本，工具自动扫描其中符合合法规则的时间戳并展示。

需要支持：

- 日志文本
- SQL 片段
- Markdown
- 接口响应片段
- 混合中英文文本

扫描规则建议：

- 匹配独立的 10 位或 13 位数字。
- 避免把长数字的一部分误判为时间戳。
- 去重展示，但保留出现次数。
- 对同一个时间戳出现多次时，可展示首次出现位置或次数。

## 页面布局

采用左右布局，以简约美观为主。

### 左侧输入区

左侧用于输入和模式切换：

- 单个时间戳输入
- JSON 粘贴区
- 普通文本粘贴区
- 自动识别模式，也可以保留手动模式切换
- 清空按钮
- 示例填充按钮

### 右侧结果区

右侧用于展示转换结果：

- 单个时间戳结果卡片
- JSON 扫描结果表格
- 文本扫描结果列表
- 合法数量、非法数量、去重数量统计
- 一键复制转换结果

桌面端保持左右布局。移动端可以改为上下布局，输入区在上，结果区在下。

## 交互细节

- 输入内容变化后自动转换，不需要用户点击提交。
- 大段文本扫描应使用防抖，避免每次输入都立即重算。
- 复制操作使用浏览器 Clipboard API。
- 对非法值给出明确但克制的提示。
- 结果为空时展示空状态，不展示空表格。
- 不使用 emoji。

## 推荐组件

- `Card`：承载输入区和结果区。
- `Textarea`：JSON 和文本粘贴。
- `Input`：单个时间戳输入。
- `Tabs`：切换输入模式。
- `Badge`：展示秒、毫秒、合法、非法状态。
- `Table`：展示扫描结果。
- `Button`：复制、清空、填充示例。
- `Toast`：复制成功或错误提示。

## 核心工具函数规划

建议放在 `apps/timestamp/src/lib` 或沉淀到 `packages/utils`：

- `detectTimestampUnit(value)`：识别秒或毫秒。
- `isValidTimestamp(value)`：判断是否为合法时间戳。
- `formatTimestamp(value)`：输出本地时间、ISO、UTC。
- `scanJsonTimestamps(value)`：递归扫描 JSON 中的时间戳。
- `scanTextTimestamps(value)`：扫描普通文本中的时间戳。
